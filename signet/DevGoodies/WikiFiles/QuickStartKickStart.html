Interim Procedure for Configuring Signet Sample Data
(aka The QuickStart KickStart)

Introduction

The Signet 1.2.0 (and 1.2.1) QuickStart package contains everything necessary to install and run Signet. However, due to some minor limitations with the QuickStart build process, the QuickStart sample database is incomplete and does not match the tutorial documentation. The following steps describe how to populate the database to match the “Roles & Data Set” sample data.


Prerequisites and Assumptions

It is assumed that the Signet QuickStart package has already been downloaded and installed on your computer.


Procedure

  1. Start the Signet QuickStart and login as 'kmart', password 'signet'

  2. Set up Dana Petri as the Subsystem Owner for Bio-X Project

    1. Select 'Act in Signet as' 'signet (admin/owner)'

    2. Select 'Switch'

    3. In 'Find A Subject', enter 'Pet' (without the quotes)

    4. Select 'Search'

    5. In 'Your search found', select the 'Petri, Dana' link

    6. In 'Designated Driver', select the 'Designate Petri, Dana' link

    7. In 'Select a Privilege Type', select 'Bio-Sciences Project X'

    8. In 'Set Conditions', select 'Effective: immediately' and 'Duration: until revoked'

    9. Select the 'Complete designation' button

    10. Select the 'My View: Current assignments to others' link to return to the main page

  3. Set up Avery Page as the Subsystem Owner for Library Management

    1. Repeat steps 2.c through 2.j, with the following substitutions

    2. At step 2.c, find subject 'apage"

    3. At step 2.e, select the 'Page, Avery' link

    4. At step 2.f, select the 'Designate Page, Avery' link

    5. At step 2.g, select 'Library Management'

    6. After step 2.j, select 'Privilege types: All' (appears just below 'Current Assignments to Others')

    7. In 'Current Privileges', select 'Current Assignments to Others'. Then select 'Show'

    At this point you have created two top-level proxies for the Signet administator: dpetri and apage. No actual assignments have been made; you have only established the ability for dpetri and apage to assign privileges within their respective subsystems. The details (and later, the history) of each privilege may be viewed by selecting the magnifier icon. Each privilege may be edited by selecting the edit link.

  4. Select 'Logout' (upper-right corner of display)

  5. Login as 'dpetri', password 'signet'

  6. Dana Petri assigns Jean Poole as a grantor and user of Bio-X Project privileges

    1. Select 'Act in Signet as' 'signet (admin/owner)'

    2. Select 'Switch'

    3. In 'Find a Subject', enter 'poo'

    4. Select 'Search' (or hit the <Enter> key on the keyboard while the cursor is in the text-entry field)

    5. In 'Your search found', select the 'Poole, Jean' link

    6. Under 'Grant a Privilege to Poole, Jean', select 'Bio-Sciences Project X' from the drop-down arrow

    7. Select the 'Start >>' button

    8. Under 'Select Privilege'', select 'Lab access via Mag Card'

    9. Select the 'Continue >>' button

    10. Under 'Select Scope', select 'Humanities & Sciences' (scroll down, if it's not visible)

    11. Select the 'Continue >>' button

    12. Under 'Select Limits', select 'Nyman Research Center'

    13. Under 'Set Conditions', select

      1. Effective: immediately

      2. Duration: until revoked

      3. Privilege holder can: use this privilege

      4. Privilege holder can: grant this privilege to others

    14. Select the 'Complete assignment' button

    15. Select 'My View: Current assignments to others'

  7. Dana Petri modifies the limits assigned to Jean Poole as a grantor and user of Bio-X Project privileges

    1. Select the 'Edit' link that appears on the row for Poole, Jean (in the 'Limits' column)

    2. Under 'Edit Limits', select both 'Higgins Laboratory' and 'Warner Laboratory'

    3. Select the 'Save changes' button

    4. Select 'Subject View [Poole, Jean]'

    5. Select the magnifier icon in the Privilege column. A popup window should appear that displays the history of this assignment.

    6. Close the popup window

    At this point Petri, the Bio-Science Project X subsystem owner, has granted lab usage privilege for 3 laboratories to Poole. The privilege starts immediately (today) and never expires. Poole may grant this privilege, or portion of it, to others.

  8. Select 'Logout'

  9. Login as 'jpoole', password 'signet'

  10. Jean Poole assigns Lee Saito some of her lab access privileges

    1. At 'change view to show...' select 'current privileges' and select the 'Show' button

    2. Under 'Find a Subject', enter '*ai*' (include the asterisks, but not the quotes)

    3. Under 'Your search found', select 'Saito, Lee'

    4. Under 'Grant a privilege' select the privilege type 'Bio-Sciences Project X'. Select the 'Start >>' button.

    5. Under 'Select Privilege' notice that the list is limited to the privilege Poole has and can grant. Select 'Lab access via Mag Card'.

    6. Select the 'Continue' button

    7. Under 'Select Scope' notice that the list is limited to the scope(s) Poole has and can grant. Select 'Biology'

    8. Select the 'Continue >>' button

    9. Under 'Select Limits' select 'Higgins Laboratory'

    10. Under 'Set Conditions' select

      1. Effective: immediately

      2. Duration: until (enter tomorrow's date, using mm-dd-yyyy format)

      3. Privilege holder can: use this privilege

      4. Privilege holder can: grant this privilege to others

    11. Select the 'Complete assignment' button

    12. Select 'My View: Current assignments to others'
